= RZ Akka library
:toc:

== Overview

RZ.Foundation.Akka is a library that simplifies integration of Akka.NET actor systems into ASP.NET applications, providing dependency injection support, base actor classes, and utility methods for actor communication.

== Installation

To install Akka in an ASP.NET project, use:

.Simple
[source,csharp]
----
builder.Services.AddAkkaSystem("SYSTEM NAME");
----

This registers the Akka actor system with the specified name into the DI container.

During Akka system startup, you can start actors by providing `ActorRegistration`.

.Start with actors
[source,csharp]
----
builder.Services.AddAkkaSystem(
    "SYSTEM NAME",
    new("test", sp => Props.Create(() => new MyActor(sp))),
    new("mySpecial", ActorFactory: (sp, system) => system.ActorOf<MySpecial>("my-special"))
    );
----

Note that `sp` (`IServiceProvider`) should not be used to resolve services during this initialization, as it may cause cyclic dependencies.

The framework also provides `IAkkaServices` (in the `RZ.Foundation.Akka` namespace) to retrieve an actor instance:

.Retrieve an actor instance
[source,csharp]
----
var akkaServices = sp.GetRequiredService<IAkkaServices>();
var myTest = akkaServices.GetService("test");
----

== Base Actor Classes

The library provides base actor classes that integrate with ASP.NET's dependency injection:

=== RzUntypedActor

Base actor class that provides dependency injection support and utility methods to simplify actor development.

[source,csharp]
----
public class MyActor : RzUntypedActor<MyActor>
{
    public MyActor(IServiceProvider sp) : base(sp)
    {
        // Services is available here
        var myService = Services.GetService<IMyService>();
    }
}
----

==== FloodAndStop

Use this to gracefully shut down your actor while ensuring all queued messages are processed before stopping.

[source,csharp]
----
FloodAndStop();
----

==== LogUnhandled

Use this to debug message handling issues by automatically logging detailed context about unhandled messages, including where they came from and which method received them.

[source,csharp]
----
LogUnhandled(message);
----

==== Run (ValueTask overload)

Use this to safely execute async operations within your actor without blocking or causing threading issues.

[source,csharp]
----
Run(message, async msg => {
    await SomeAsyncOperation(msg);
});
----

==== Run (Outcome overload)

Use this when you need automatic error logging for failed operations - errors are logged with full context about the message and failure.

[source,csharp]
----
Run(message, async msg => {
    return await ProcessWithOutcome(msg);
});
----

==== OnFinalization

Override this to customize how your actor handles messages during shutdown, useful for cleanup or special termination logic.

[source,csharp]
----
protected override void OnFinalization(object message)
{
    // Custom shutdown handling
    base.OnFinalization(message);
}
----

== Solid Message Passing

The library provides a safe way to ask and retrieve data from an actor. A message may implement
`CanResponse<T>`, where `T` is the type of response expected from the message.

Note that this framework relies on the `Outcome<T>` type from the `RZ.Foundation` library, which provides Go-like error handling.

For example, if you have a message `GetCurrentPrice` and expect a `decimal` return, the message would look like this:

[source,csharp]
----
public sealed record GetCurrentPrice : CanResponse<decimal>;

IActorRef myAccount; // and you have an account actor
----

=== Query

To ask for the price, it looks like this:

[source,csharp]
----
if (Fail(await new GetCurrentPrice().RequestTo(myAccount), out var error, out var amount)) {
    Console.WriteLine("Unexpected error: {0}", error);
}
else {
    Console.WriteLine("My amount is {0}", amount);  // amount is `decimal` type
}
----

=== Actor message handling

[source,csharp]
----
public sealed class MyAccountManager : UntypedActor
{
    decimal? amount;

    public override void Receive(object message)
    {
        if (message is GetCurrentPrice m)
            if (amount is null)
                m.RaiseError(Sender, new ErrorInfo("not-set", "Amount is not yet set"));
            else
                m.Respond(Sender, amount);  // strong-typed check!
        else
            UnhandleMessage(message);
    }
}
----
